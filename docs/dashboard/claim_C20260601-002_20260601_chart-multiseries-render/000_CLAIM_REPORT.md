**claim id**: C20260601-002
**実施日**: 2026-06-01
**対象機能**: dashboard（チャート表示。共通 `src/components/MetricChart.tsx` + collection/runner 横断）
**緊急度推定**: high（旗艦チャート「直近30日の推移」が 2 サービス以上で実質機能不全。データ自体は無事＝表示層バグ）

## 1. クレーム原文

```
2つのサービスを登録したのでチャートに折れ線が２本表示されることを期待するが、
2つの値になった時点で線が描画されなくなっている
しかも同じ時間帯のデータがx軸上でずれてプロットされているようだ
時間軸でミリ秒まで表示しているので見づらい。分まででよい
```

スクショ（service-hub dashboard「直近30日の推移」ユーザー数(count)）:
凡例＝花メモ(緑)/防災バッグチェッカー(青)。緑のみ y=8 のフラット線、青は未描画。
x 軸ラベルが `2026-05-28T05:07:31.725Z` 等の ISO 文字列（ミリ秒）。
ヘッダ「2 up・0 down」「直近: 2 サービス / エラー 0 件 (ok)」= pull は成功（404 は別件 C20260601-001 で解消）。

## 2. 分解結果

### 2.1 期待挙動 (Expected)
- 2 サービス登録 → 折れ線が **2 本**重ね描きされる。
- 同じ時間帯のデータは x 軸上の**同じ位置**にプロットされる。
- 時間軸ラベルは**分単位まで**（ミリ秒は不要、見やすく）。

### 2.2 現実挙動 (Actual)
1. **2 つの値（2 サービス）になった時点で線が描画されなくなる**（1 本も繋がらない/途切れる）。
2. **同じ時間帯のデータが x 軸上でずれて**プロットされる。
3. **x 軸がミリ秒まで**の ISO 文字列表示で見づらい。

### 2.3 発生条件
- 操作: dashboard を開く（2 サービス登録済み・pull 成功状態）。
- 機構（調査済み、深掘りは fix 側）:
  - `src/components/MetricChart.tsx` の `XAxis` が `dataKey="capturedAt"` のまま **カテゴリ軸**（`type="number"`+時間スケール未使用）→ 実時間間隔を無視し各 capturedAt を等間隔配置 → **x 軸ずれ**。XAxis に `tickFormatter` が無く ISO 文字列をそのまま表示 → **ミリ秒表示**。
  - `src/features/collection/runner.ts:57` が SnapshotRow ごとに `capturedAt: now().toISOString()` を生成 → **同一 collection run でもサービス/メトリクスごとに別タイムスタンプ**。
  - `MetricChart.tsx:62-76` `mergeSeries` は capturedAt **完全一致**でのみ行マージ → 2 サービスの同一 run の点が別行になり、各 series に **null が交互混入** → recharts `Line` の `connectNulls` 既定 false（`MetricChart.tsx:151-161` で未指定）→ **線が途切れる**。
  - 結果: 1 サービス時は 1 本描けるが、2 サービスになると相互の null で両線とも途切れ、x 位置もずれる。

### 2.4 影響範囲
- 該当: 2 サービス以上を登録した全 dashboard 閲覧（= 本来の中核 UC「service 横断比較」）。
- 業務影響: multi-service 重ね描き比較が機能しない（service-hub の主要価値の一つ）。
- データ影響: なし（snapshot データは正常、表示層のみ）。
