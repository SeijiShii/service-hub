## 1. 症状
service-hub dashboard「直近 30 日の推移」チャート（ユーザー数等）で:
1. 2 サービス（2 値）になった時点で折れ線が途切れ、描画されない。
2. 同じ時間帯のデータが x 軸上でずれてプロットされる。
3. x 軸ラベルが `2026-05-28T05:07:31.725Z` のように**ミリ秒の ISO 文字列**で見づらい。

スクショ: 凡例＝花メモ(緑)/防災バッグチェッカー(青)。緑のみ y=8 フラット、青未描画。「2 up・エラー 0 件 (ok)」。

## 2. 期待挙動
基準 SPEC `revise_timeseries-topchart_20260528/001_REVISE_SPEC.md §16/30/31`:
「全 active service を 1 枚の chart に**重ね描き**（service 別色）、過去 30 日」。
→ 2 サービスなら 2 本の折れ線、同一時刻は同 x 位置、軸は読みやすい粒度（分まで）。

## 3. 再現手順
1. 2 サービス（active）を登録し、service-info / 各 provider が正常 pull される状態にする。
2. `/api/cron/collect`（または dashboard「今すぐ pull」）を複数回（複数日相当）実行し snapshot を蓄積。
3. dashboard を開き「直近 30 日の推移」チャートを見る。
→ 2 サービス分の点が現れる区間で線が途切れ、x がずれ、軸ラベルがミリ秒表示。

## 4. 再現環境
| 項目 | 値 |
|---|---|
| 画面 | service-hub dashboard 上部 time-series chart |
| 描画 | recharts `LineChart`（`src/components/MetricChart.tsx`） |
| データ | usage_snapshots（複数 service × 複数 run） |
| 条件 | active service ≥ 2、各 run で複数行が別 `new Date()` を取得 |

## 5. 影響範囲
- 影響: 2 サービス以上を登録した全 dashboard 閲覧（= 中核 UC「service 横断比較」）。
- 業務影響: multi-service 重ね描き比較が機能しない。
- データ影響: **なし**（usage_snapshots は無事、表示・整列のみの不具合）。
- セキュリティ影響: なし。

## 6. 暫定回避策
- service-detail（単一 service）チャートは 1 series のため概ね描画可（軸ミリ秒表示の見づらさは残る）。
- 根本回避策はなし → 本 fix で対応。

## 10. タイムラインから抽出した仮説
- 仮説1（確定）: `runner.ts` の per-row `capturedAt` 生成 → 同一 run の複数 service 点が別タイムスタンプ → `mergeSeries` 完全一致マージで非整列。
- 仮説2（確定）: `XAxis` がカテゴリ軸 + tickFormatter 無し → x 等間隔配置（ずれ）+ ISO 生表示（ミリ秒）。
- 仮説3（確定）: `<Line connectNulls>` 既定 false → 非整列で生じた null 交互により線が途切れる。
