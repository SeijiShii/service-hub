## 1. 症状
ログイン済みの seiji 本人でも、ダッシュボード（`/api/dashboard/summary`）とサービス詳細（`/api/services/[slug]/timeseries`）が **常に 401** を返し、画面がデータを表示できない。

## 2. 期待挙動
Clerk でログインした seiji（`ALLOWED_USER_ID` に一致）のリクエストは 200 でデータを返し、未認証は 401、別ユーザーは 403。

## 3. 再現手順
1. 実キー（Clerk test 鍵 + `ALLOWED_USER_ID`）を設定して本番/preview で起動。
2. seiji でログイン後、ダッシュボードを開く。
3. `GET /api/dashboard/summary` が 401 を返し、画面は error 状態。

| 項目 | 値 |
|---|---|
| 構成 | Vite SPA + Vercel Functions（Next.js ではない） |

## 5. 影響範囲
- 影響ユーザー: 全員（唯一のユーザー seiji 含む）
- 影響機能: 認可が要る全 API（dashboard / service-detail）。cron は `CRON_SECRET` 別系統で影響なし。
- データ影響: なし　/　**本番影響: なし（デプロイ前に `/flow:release` Phase2 検証で検出）**
