## 背景・経緯

現仕様（D20260526-004）では新マイクロサービス追加のたびに HUB 側で `services.toml`（Git SoT）と `.env`（Vercel Secrets: 旧 `HANAMEMO_CLERK_SECRET` / `HANAMEMO_HUB_SECRET`）の両方を書き換え、再デプロイが必要だった。seiji が「サービスが増えるたびの再デプロイ手間を解消したい」と要望。

当初案は「HUB に POST 登録 API + 全サービス共通鍵」だったが、設計検討で以下を確認:
- 再デプロイの真因は descriptor だけでなく**サービス固有シークレットの供給**でもある → POST だけでは `.env` 追記が残り痛みが半分しか消えない（[D20260528-002] が必要）。
- 単一共有鍵の公開 POST は爆発半径大（1 漏洩で全サービス登録を上書き可能）+ write 経路 SSRF + Clerk ゲートへの公開穴。単一ユーザー内部ツールには過剰リスク。
→ seiji 判断「DB 化でよい」。公開 POST/共通鍵は不採用とし、**DB SoT + Clerk ゲート内 admin write**（seiji が HUB 画面で 1 件登録）を採用。
