## 学習・改善
- cross-repo の metric 契約は、producer の機能名 (tip=投げ銭) ではなく consumer 横断で意味が一貫する**汎用ドメイン名 (revenue)** を canonical にすべき。源泉 (寄付/売上/投げ銭) はサービス差で、契約はそれを抽象した収益で持つ。
- 契約キー改名の cross-repo コストは、consumer 側 adapter の**後方互換エイリアス**で吸収すると producer の同期再デプロイが不要になり、安全に移行できる (既存本番申告を壊さない)。
