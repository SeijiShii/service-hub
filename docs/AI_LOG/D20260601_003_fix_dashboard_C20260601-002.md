**主要決定**:
- 根本原因 = 「1 run = 単一 capturedAt」不変条件の未定義。runner.ts:57 が per-row `new Date()` 生成（api/cron/collect が now 未注入）→ 同一 run の service 間で時刻ずれ → MetricChart 完全一致マージ/カテゴリ軸/connectNulls 既定で線断裂・x ずれ・ミリ秒表示。
- テストが固定 now で per-row ドリフトを隠蔽していた（runner.test.ts:40）。
- 修正 = runner で startedAt を全行共有 + MetricChart の x 連続時間軸化(分整形) + mergeSeries 正規化 + connectNulls 再評価。service-detail 後方互換を回帰確認。
**依存**: claim C20260601-002 (D20260601-002-05 bug 判定) / revise timeseries-topchart (SPEC) / revise biz-charts
**生成・更新ファイル**: fix サブフォルダ 000-004 + INDEX、dashboard INDEX (claim commit で追加済)
**学習・改善**: 「テスト GREEN ≠ 本番正常」の典型 — 時刻系は固定 now でなく可変クロックで non-determinism を再現すべき。producer(capturedAt 生成) と consumer(chart x 軸前提) の不変条件不整合は claim→fix の 5 Whys で炙り出せた。
